# A8 四流程填报信息清单

版本：v1.0.0  
边界：只生成 A8 保存待发草稿，不发送。

## 通用输入

| 项目 | 说明 |
| --- | --- |
| A8账号密码 | 只放当前会话或当前 PowerShell 环境变量：`SEEYON_USERNAME` / `SEEYON_PASSWORD`。 |
| 输入文件 | 推荐使用 `A8_ASSISTANT_INPUT_JSON_FILE` 指向 UTF-8 JSON。 |
| 运行模式 | `A8_TRACE_LEVEL=fast/normal/debug`，也可用 `A8_RUN_MODE`。 |
| 保存策略 | 保存前做页面回读校验；不点击发送。 |

## 1. 集成项目创建单

入口：`npm run assist:project-create-save`  
模式：`fast` / `normal` / `debug`

需要用户提供：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `projectName` | 项目名称 | 是 |
| `customerName` | 客户名称，A8 客户关系控件需能匹配 | 是 |
| `sales` | 销售人员显示名或可搜索关键词 | 是 |
| `business` | 商务人员显示名或可搜索关键词 | 是 |
| `presales` | 售前人员显示名或可搜索关键词 | 是 |
| `winRate` | 项目成单率，可选值：`25%` / `50%` / `75%` / `100%` | 是 |
| `progress` | 项目进度，例如 `前期洽谈`、`方案介绍和商务报价`、`项目已成单`、`项目搁置` | 是 |
| `estimatedAmount` | 项目预估金额 | 是 |
| `province` | 项目所属省 | 是 |
| `city` | 项目所属市 | 是 |
| `district` / `county` | 项目所属区县 | 建议填 |
| `projectDescription` | 项目描述 | 可选 |

示例：

```json
{
  "projectName": "<项目名称>",
  "customerName": "<客户名称>",
  "sales": "<销售>",
  "business": "<商务>",
  "presales": "<售前>",
  "winRate": "100%",
  "progress": "项目已成单",
  "estimatedAmount": "800000",
  "province": "<省>",
  "city": "<市>",
  "district": "<区县>",
  "projectDescription": "<项目描述>"
}
```

## 2. 集成项目立项申请

入口：`npm run assist:init-apply-save`  
模式：`fast` / `normal` / `debug`

需要用户提供：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `relatedProjectCreateCode` | 关联的项目创建单编号；也可用 `A8_LINKED_CREATE_NO` | 建议填 |
| `buildUnit` | 建设单位 | 按流程需要 |
| `constructionUnit` | 施工单位 | 按流程需要 |
| `needDelivery` | 是否需要交付，常用默认：`是` | 可默认 |
| `priceSystem` | 价格体系，常用默认：`战略合作价` | 可默认 |
| `softwareIntegrationConfirm` | 软件集成确认，例如 `已确认-具备对接条件` | 可默认 |
| `regulatorDemand` | 监管需求，可为字符串或数组，常用默认：`无` | 可默认 |
| `businessType` / `business` | 所属业务，常用默认：`AI安全事业部` | 可默认 |
| `industryType` / `industry` | 所属行业，常用默认：`基建数字化行业业务` | 可默认 |
| `materials` / `rows` | 明细行数组 | 是 |

明细行：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `code` / `materialNo` / `productNo` | 料号/产品编号；空料号会保留并告警 | 建议填 |
| `qty` / `quantity` | 数量 | 建议填 |
| `taxPrice` / `unitPrice` / `price` | 含税单价 | 建议填 |
| `sourceRow` | 来源行号，便于回溯 | 可选 |
| `label` | 行说明 | 可选 |

示例：

```json
{
  "relatedProjectCreateCode": "<项目创建单编号>",
  "needDelivery": "是",
  "priceSystem": "战略合作价",
  "softwareIntegrationConfirm": "已确认-具备对接条件",
  "regulatorDemand": "无",
  "businessType": "AI安全事业部",
  "industryType": "基建数字化行业业务",
  "materials": [
    { "sourceRow": 1, "code": "<料号>", "qty": "2", "taxPrice": "420" }
  ]
}
```

## 3. 报价清单变更

入口：`npm run assist:change-existing-save`  
模式：`fast` / `normal` / `debug`

需要用户提供：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `projectNameKeyword` / `projectName` | 用于搜索项目的关键词 | 是 |
| `projectNo` / `projectNumber` | 项目编号；可填 `AUTO` 让脚本在关键词唯一匹配时自动取号 | 建议填 |
| `accountName` / `account` | 账套 | 按流程需要 |
| `businessName` / `business` | 所属业务，常用默认：`AI安全事业部` | 可默认 |
| `industryName` / `industry` | 所属行业，常用默认：`基建数字化行业业务` | 可默认 |
| `changeType` | 行变更类型，常用：`新增` / `变更` | 是 |
| `changedAuxMaterial` | 变更后辅料，常用默认：`0` | 可默认 |
| `changedInstallDebugFee` | 变更后安装调试费，常用默认：`0` | 可默认 |
| `deliveryFee` / `estimatedDeliveryServiceFee` | 预估交付实施服务费，常用默认：`0` | 可默认 |
| `materials` / `rows` | 变更明细行数组 | 是 |

明细行：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `code` / `materialNo` / `productNo` | 产品/料号；非空时按精确编码选择 | 建议填 |
| `qty` / `quantity` / `addedQty` | 新增数量或变更数量；负数表示减少 | 建议填 |
| `taxPrice` / `unitPrice` / `price` | 含税单价；不填时尽量保留原值或按流程告警 | 可选 |
| `changeType` | 单行变更类型；为空时继承主 `changeType` | 可选 |
| `sourceRow` | 来源行号 | 可选 |

示例：

```json
{
  "projectNameKeyword": "<项目关键词>",
  "projectNo": "AUTO",
  "accountName": "<账套>",
  "businessName": "AI安全事业部",
  "industryName": "基建数字化行业业务",
  "changeType": "新增",
  "changedAuxMaterial": "0",
  "changedInstallDebugFee": "0",
  "deliveryFee": "0",
  "materials": [
    { "sourceRow": 1, "code": "<产品编号>", "qty": "1", "taxPrice": "100" }
  ]
}
```

## 4. 存货档案新建申请-批量

入口：`npm run assist:inventory-batch-save`  
模式：`normal` / `debug`，`fast` 明确不开放。

需要用户提供：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `businessDepartment` / `department` | 主表事业部门，常用默认：`AI安全生产事业部` | 是 |
| `items` | 存货明细行数组 | 是 |

明细行：

| JSON字段 | 说明 | 是否必填 |
| --- | --- | --- |
| `materialCode` / `code` / `materialNo` | 物料编码；新建申请可为空，保存后需人工复核 | 可选 |
| `productName` / `name` | 产品名称 | 是 |
| `model` / `specification` / `spec` | 规格型号 | 建议填 |
| `invoiceModel` / `invoiceSpec` | 开票规格型号；默认可取规格型号 | 是 |
| `brand` | 品牌 | 是 |
| `cost` | 成本 | 是 |
| `applicationCategory` | 申请类别，常用默认：`外购` | 可默认 |
| `researchMethod` | 研发方式，常用默认：`外购` | 可默认 |
| `subsystem` | 子系统-2025，常用默认：`其他子系统` | 可默认 |
| `productSeries` | 产品系列-2025，常用默认：`通用产品` | 可默认 |
| `materialSegment` | 料号段，常用默认：`WG_LS` | 可默认 |
| `originalModel` | 原厂型号，默认可取开票规格型号 | 可默认 |
| `manufacturerName` | 厂家名称，默认可取品牌 | 可默认 |
| `designatedSupplier` | 是否指定供应商，常用默认：`否` | 可默认 |
| `temporaryMaterial` | 是否临时料号，常用默认：`是` | 可默认 |
| `bomType` | BOM 类型，常用默认：`基线` | 可默认 |
| `unit` | 计量单位，常用默认：`pcs` | 可默认 |
| `tagType` | 标签类型，常用默认：`基线` | 可默认 |
| `outputTaxRate` | 销项税率，常用默认：`13%` | 可默认 |
| `inputTaxRate` | 进项税率，常用默认：`13%` | 可默认 |
| `promotionAdvice` | 推广建议，常用默认：`不推荐` | 可默认 |
| `freight` | 运输费用，常用默认：`0` | 可默认 |
| `totalCost` | 成本合计，默认可取成本 | 可默认 |
| `channelPrice` / `strategicPrice` / `directPrice` | 渠道价、战略合作价、直销价 | 可选 |
| `remark` | 备注 | 可选 |

示例：

```json
{
  "businessDepartment": "AI安全生产事业部",
  "items": [
    {
      "sourceRow": 1,
      "productName": "<产品名称>",
      "model": "<规格型号>",
      "invoiceModel": "<开票规格型号>",
      "brand": "<品牌>",
      "cost": "100",
      "applicationCategory": "外购",
      "researchMethod": "外购",
      "subsystem": "其他子系统",
      "productSeries": "通用产品",
      "materialSegment": "WG_LS",
      "outputTaxRate": "13%",
      "inputTaxRate": "13%"
    }
  ]
}
```
