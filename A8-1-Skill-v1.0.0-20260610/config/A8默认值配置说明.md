# A8 默认值配置说明

版本：v1.0.0  
用途：给人和 Codex 统一理解“哪些值可以默认补、哪些必须由用户或材料提供”。

## 默认值优先级

1. 用户在当前需求里明确提供的值。
2. 用户给的 Excel、JSON、Word、Markdown 等材料里的值。
3. 本文件记录的默认值。
4. 无可靠默认值时保留空值，并在结果里提示人工复核。

## 正式运行前填写方法

正式运行前建议采用“两层默认值”：

1. `default-values.template.json` 作为外发模板，只放通用建议值，不写账号、密码、验证码、A8 草稿 ID 或真实项目材料。
2. 使用方第一次运行前复制一份本地文件：`config/default-values.local.json`。
3. 人工打开 `A8默认值配置说明.xlsx`，在“默认值总表”和各流程 sheet 的“建议修改为”列填写现场默认值。
4. 把确认后的默认值同步到 `default-values.local.json`。正式运行前由 Codex 或操作者优先读取 local 文件并合并到本次输入 JSON；template 只作为兜底参考。
5. 没把握的字段不要硬填，保持空值或标记为不启用，让运行前校验提醒人工补充。

运行时合并逻辑：

| 优先级 | 来源 | 处理方式 |
| --- | --- | --- |
| 1 | 用户当前明确给出的值 | 直接使用。 |
| 2 | 用户提供的材料文件 | 从 Excel、JSON、Word、Markdown 等材料中解析。 |
| 3 | `default-values.local.json` | 作为现场已确认默认值。 |
| 4 | `default-values.template.json` | 只作为通用兜底。 |
| 5 | 空值 | 关键字段缺失时阻断；非关键字段保留空值并告警。 |

关键字段不允许默认硬填，例如项目名称、客户、销售/商务/售前、金额、项目定位信息、料号、数量、单价、存货名称、型号、品牌、成本等。可默认字段主要是保存策略、运行模式、业务/行业、税率、申请类别、研发方式、计量单位、BOM 类型、费用为 0 的业务默认项等。

## 通用默认

| 配置项 | 默认/建议值 | 说明 |
| --- | --- | --- |
| 保存策略 | 保存待发 | 不发送。 |
| 登录策略 | HTTP 优先，失败后无头 Chrome 表单登录 | 验证码走截图和一次性输入。 |
| 项目创建运行模式 | `fast` | 失败或不明确时升级 `normal` / `debug`。 |
| 项目立项运行模式 | `fast` | 失败或不明确时升级 `normal` / `debug`。 |
| 报价变更运行模式 | `fast` | 失败或不明确时升级 `normal` / `debug`。 |
| 存货批量运行模式 | `normal` | 仅支持 `normal` / `debug`，不开放 `fast`。 |

## 集成项目创建单

| 字段 | 默认值 | 建议 |
| --- | --- | --- |
| 项目名称 | 无默认 | 必须来自用户。 |
| 客户名称 | 无默认 | 必须来自用户，且需 A8 客户关系控件可匹配。 |
| 销售 / 商务 / 售前 | 无默认 | 必须来自用户或项目资料，不在发布包内硬编码人员。 |
| 项目成单率 | 可按业务约定填 `100%` | 若用户未说，不建议盲填。 |
| 项目进度 | 可按业务约定填 `项目已成单` | 若用户未说，不建议盲填。 |
| 项目预估金额 | 无默认 | 必须来自用户或材料。 |
| 省 / 市 / 区县 | 无默认 | 建议用户明确给出。 |

## 后续可机器化

`default-values.template.json` 是给后续运行时读取配置预留的结构。v1.0.0 先把默认值作为人工和 Codex 可读文档管理，由运行前预检合并进本次输入 JSON，不把它作为脚本强制自动读取的运行时配置源。
